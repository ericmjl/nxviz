# Version v0.8.0

This release introduces significant enhancements, including multi-backend support with Plotly for interactive visualizations, alongside various improvements in code quality and CI workflows. The update also modernizes the repository to align with the latest development standards.

## New Features

- Added multi-backend support with a new Plotly interactive backend, allowing users to switch backends easily for enhanced visualization capabilities. This includes interactive features such as hover, zoom, and pan. The update also introduces a backend abstraction layer and several new tests to ensure robustness. (73e349) (Eric Ma)

## Bug Fixes

- Corrected the use of the PYPI_API_TOKEN for publishing in the CI workflow to ensure secure and successful package deployment. (9cc081) (Eric Ma)
- Fixed the parsing of bumped versions from the uv output in the CI workflow, ensuring accurate version management. (c7087c) (Eric Ma)
- Removed unused variables flagged by the ruff linter, improving code cleanliness and maintainability. (452c02) (Eric Ma)
- Resolved pre-commit hook failures by updating dependencies and excluding certain directories from checks, ensuring smoother development workflows. (acfcf8) (Eric Ma)
- Reverted changes to notebooks in the CI to maintain consistency and prevent unintended modifications. (046fb2) (Eric Ma)
- Removed broken interrogate and pydoclint hooks from the CI, addressing compatibility issues with Python 3.12+ and reducing violations. (9fba08) (Eric Ma)

## Deprecations

- Removed support for Python 3.10 and 3.11, focusing testing on Python 3.12 and 3.13 to streamline the CI process and align with future Python releases. (e3709f) (Eric Ma)

This release marks a significant step forward in enhancing the flexibility and maintainability of the project, with a focus on modern development practices and interactive visualization capabilities.
