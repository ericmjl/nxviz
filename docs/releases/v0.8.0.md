## Version 0.8.0

This release adds multi-backend support to nxviz, enabling interactive plotting backends alongside the existing matplotlib renderer.

### New Features

- Added `PlotBackend` protocol and `get_backend()` factory for pluggable rendering backends.
- Added `backend="plotly"` parameter to all plot functions for interactive hover/zoom/pan plots.
- Added `nxviz/paths.py` for backend-agnostic edge path coordinate computation.
- Added `nxviz/backends/matplotlib_backend.py` wrapping existing matplotlib rendering.
- Added `nxviz/backends/plotly_backend.py` with interactive Plotly backend.
- All six plot types supported: circos, arc, hive, matrix, parallel, geo.
- Optional plotly dependency via `pip install nxviz[plotly]`.
- Added `AGENTS.md` with coding standards and codebase overview.

### Design Documentation

- High-level design document for multi-backend support.
- Low-level design document with module interfaces.
- Five EARS requirement specifications (protocol, matplotlib, plotly, paths, API).

### Bug Fixes

- Fixed CI workflow to correctly parse bumped version from `uv version` output.
- Fixed CI workflow to use `PYPI_API_TOKEN` for PyPI publishing.

### Other Changes

- Added `explore` pixi feature with marimo and plotly for interactive development.
- Updated architecture documentation with backend system section.
- 44 new tests covering backends, path computation, and plotly rendering.
